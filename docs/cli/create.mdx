## Overview

The `cpm create` command group creates new projects and shared components.

```bash
cpm create project <NAME> [OPTIONS]   # Create a project
cpm create skill <NAME> [OPTIONS]     # Create a shared skill
cpm create agent <NAME> [OPTIONS]     # Create a shared agent
cpm create hook <NAME> [OPTIONS]      # Create a shared hook
cpm create rule <NAME> [OPTIONS]      # Create a shared rule
```

---

## cpm create project

Creates a new project in the mono repo with the standard Claude Code structure.

## Arguments

| Argument | Description |
|----------|-------------|
| `NAME` | Name for the new project |

## Options

| Option | Short | Description |
|--------|-------|-------------|
| `--description` | `-d` | Project description |
| `--skills` | `-s` | Comma-separated list of shared skills to add |
| `--agents` | `-a` | Comma-separated list of shared agents to add |

## Examples

### Basic project

```bash
cpm create project my-app
```

### With description

```bash
cpm create project my-app -d "My application project"
```

### With initial dependencies

```bash
cpm create project my-app --skills code-review,testing --agents debugger
```

### Short form

```bash
cpm create project my-app -d "My app" -s skill1,skill2 -a agent1
```

## Created Structure

```
projects/my-app/
├── project.json        # Project manifest
├── CLAUDE.md           # Project instructions
├── .claude/
│   ├── settings.json   # Claude settings
│   ├── skills/         # Skills directory
│   ├── agents/         # Agents directory
│   ├── hooks/          # Hooks directory
│   └── rules/          # Rules directory
└── outputs/            # Project outputs
```

## Generated Files

### project.json

```json
{
  "name": "my-app",
  "description": "My application project",
  "dependencies": {
    "skills": [],
    "agents": [],
    "hooks": [],
    "rules": []
  }
}
```

### CLAUDE.md

```markdown
# my-app

My application project

## Project Structure

This is a Claude Code project managed by CPM.

## Guidelines

- Follow project conventions
- Use available skills and agents
```

<Note>
If you specify `--skills` or `--agents`, CPM will automatically run `cpm sync` to create the symlinks.
</Note>

## cpm create skill

Creates a shared skill in the `shared/skills/` directory.

### Usage

```bash
cpm create skill <NAME> [OPTIONS]
```

### Options

| Option | Short | Description |
|--------|-------|-------------|
| `--description` | `-d` | Skill description |
| `--skills` | `-s` | Comma-separated skill dependencies |
| `--hooks` | `-h` | Comma-separated hook dependencies |
| `--rules` | `-r` | Comma-separated rule dependencies |

### Examples

```bash
# Basic skill
cpm create skill code-review

# With description
cpm create skill code-review -d "Code review assistant"

# With dependencies
cpm create skill advanced-review --skills base-review,utils --rules security
```

### Created Structure

```
shared/skills/code-review/
├── SKILL.md      # Skill instructions
└── skill.json    # Metadata and dependencies
```

## cpm create agent

Creates a shared agent in the `shared/agents/` directory.

### Usage

```bash
cpm create agent <NAME> [OPTIONS]
```

### Options

| Option | Short | Description |
|--------|-------|-------------|
| `--description` | `-d` | Agent description |
| `--skills` | `-s` | Comma-separated skill dependencies |
| `--agents` | `-a` | Comma-separated agent dependencies |
| `--hooks` | `-h` | Comma-separated hook dependencies |
| `--rules` | `-r` | Comma-separated rule dependencies |

### Examples

```bash
# Basic agent
cpm create agent debugger

# With description
cpm create agent debugger -d "Debugging assistant"

# With full dependencies
cpm create agent security-audit \
  -d "Security audit agent" \
  --skills vuln-scan,code-review \
  --hooks pre-commit \
  --rules security
```

### Created Structure

```
shared/agents/security-audit/
├── AGENT.md      # Agent instructions
└── agent.json    # Metadata and dependencies
```

## Component Metadata

All created components include a metadata JSON file with this structure:

```json
{
  "name": "component-name",
  "description": "Optional description",
  "dependencies": {
    "skills": ["skill-a", "skill-b"],
    "agents": ["agent-x"],
    "hooks": ["hook-y"],
    "rules": ["rule-z"]
  }
}
```

When a component with dependencies is added to a project via `cpm add`, all dependencies are automatically installed.
