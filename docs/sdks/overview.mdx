## Choosing an SDK

<Tabs>
  <Tab title="Python">
    **Best for:**
    - Quick setup and prototyping
    - Integration with Python-based tools
    - Users familiar with pip/pipx

    ```bash
    pip install cpm
    ```
  </Tab>
  <Tab title="TypeScript">
    **Best for:**
    - Node.js projects
    - Frontend/fullstack developers
    - Integration with npm ecosystem

    ```bash
    npm install cpm
    ```
  </Tab>
  <Tab title="Go">
    **Best for:**
    - Single binary distribution
    - System-level integrations
    - Performance-critical environments

    ```bash
    # Coming soon
    go install cpm
    ```
  </Tab>
  <Tab title="Rust">
    **Best for:**
    - Maximum performance
    - Memory-constrained environments
    - Embedding in other applications

    ```bash
    # Coming soon
    cargo install cpm
    ```
  </Tab>
</Tabs>

### SDK Implementation Guide

Each SDK should implement:

1. **Core modules:**
   - Config loading/saving
   - Project resolution
   - Symlink management

2. **CLI commands:**
   - init, create, add, remove, link, unlink, get, clone, sync

3. **Tests:**
   - Unit tests for all modules
   - Integration tests for CLI commands
